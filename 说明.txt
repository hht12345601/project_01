一，git init 创建git仓库
二，git add 文件名  增加到git管理 暂存区域
三，git status -s 查看状态
四，git commit -m "提交了什么" 提交到git仓库
modifie已修改
五，修改后需要再次git add 
git add 功能：
1，开始跟踪新文件
2，把已经跟踪，且修改的文件放到赞存区
3，把有冲突的文件标记为已解决的状态
撤销对文件的修改  git checkout -- index.html
六，向赞存区一次性提交多个文件
git add .  把所有新增和修改的增添到暂存区
七，取消已暂存的文件
git reset HEAD 移除的文件名称 git reset HEAD . 移除全部暂存文件
八，跳过暂存区 直接提交到git仓库
git commit -a -m "描述消息"
九，移除文件
1 git rm -f 文件名 同时从仓库和工作区移除
2git rm --cached 文件名 只从git中移除 保留工作区
十，忽略文件gitignore
以#开头是注释
以/结尾是目录
以/开头是防止递归
以！开头是表示取反
glob模式进行文件和文件夹的匹配（简化的正则表达式）
*表示匹配0个或多个任意字符
[abc]表示匹配任意一个在列的字符
？表示只匹配一个任意字符
[0-9]表示俩范围内都可以匹配
**表示匹配任意中间目录
